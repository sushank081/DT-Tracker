# 🗄️ Database Architecture & Control Ledger

Welcome to the core data layer of the **Downtime Core Telemetry Engine**. This repository segment houses the relational database schemas, structural mapping models, and authentication workflows running on **PostgreSQL**. 

Our database layer acts as the absolute source of truth—capturing real-time telemetry metrics, logging industrial downtime tickets across plant shop units, and enforcing cryptographic access control for our shift operators.

---

## 🚀 Quick Start & Database Setup

To provision and initialize this data engine locally or across a factory floor environment, execute the structural migrations sequentially.

### 1. Environmental Prerequisites
Ensure your target environment is running **PostgreSQL 14+** and your database user possesses full privileges over the target tablespace.

### 2. Schema Deployment
Execute the structural definitions inside your preferred PostgreSQL management interface (e.g., pgAdmin or psql terminal):

```sql
-- Initialize User Registry Matrix
CREATE TABLE IF NOT EXISTS public.ticket_master_user
(
    id integer NOT NULL GENERATED BY DEFAULT AS IDENTITY ( INCREMENT 1 START 1 MINVALUE 1 MAXVALUE 2147483647 CACHE 1 ),
    name character varying(50) NOT NULL,
    created_date timestamp without time zone NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_date timestamp without time zone,
    is_active character varying(10) DEFAULT 'ACTIVE'::character varying,
    user_name character varying(50),
    user_pwd character varying(255), -- Allocated 255 width to securely house Bcrypt signatures
    email character varying(50),
    CONSTRAINT ticket_master_user_pkey PRIMARY KEY (id)
);

ALTER TABLE IF EXISTS public.ticket_master_user OWNER to postgres;
