# 📖 Database Data Catalog & Data Dictionary

This document serves as the absolute data dictionary for the **Downtime Core Telemetry Engine (DT-Tracker)** PostgreSQL schema. It outlines the structural mapping, column definitions, data types, and relational foreign key configurations generated by the system's ERD.

---

## 🗺️ Schema Overview

The database uses a robust **Star Schema** variant optimized for transactional speed and deep analytical querying:
* **Fact Table:** `ticket_app_downtime_tracker` (captures live manufacturing floor incidents).
* **Dimension/Master Tables:** 8 supporting master registries controlling users, shops, shifts, statuses, priorities, and categorizations.

---

## 📊 Fact Table Dictionary

### 1. `ticket_app_downtime_tracker`
Stores all logged manufacturing floor downtime incidents, actions taken (RCA/ICA/PCA), and chronological timestamps.

| Column Name | Data Type | Key / Constraint | Description |
| :--- | :--- | :---: | :--- |
| `id` | bigint | **PK**, Identity | Unique auto-incrementing transactional identifier for the incident ticket. |
| `internal_ticket` | varchar(50) | Not Null | System-generated internal tracker logging code sequence. |
| `month_name` | varchar(20) | Nullable | Text name of the month when the incident occurred (e.g., "June"). |
| `service_ticket` | varchar(50) | Nullable | External IT or infrastructure service desk cross-reference ticket number. |
| `ticket_raiser_id` | integer | **FK** | Links to `ticket_master_user.id` indicating who initiated the log entry. |
| `ticket_raiser` | varchar(50) | Not Null | Captured display name or terminal identifier of the ticket creator. |
| `issue_occurance` | integer | Not Null, Default `0` | Recurrence counter tracking frequency behavior of this specific line fault. |
| `ticket_type_id` | integer | **FK** | Links to `ticket_master_type.id` grouping the core ticket context. |
| `ticket_type` | varchar(50) | Nullable | Redundant display string tracking the core category type of the ticket. |
| `shop_id` | integer | **FK** | Links to `ticket_master_shops.id` mapping the physical shop floor zone. |
| `shop` | varchar(50) | Nullable | Cached display name of the shop department layout matrix. |
| `ticket_date` | date | Not Null | Calendar date when the downtime event occurred. Used for primary chronological sorting. |
| `ticket_time` | time | Not Null | Microsecond-accurate timestamp showing when the stop occurred. Used for sub-sorting. |
| `shift_id` | integer | **FK** | Links to `ticket_master_shift.id` identifying the operational workforce window. |
| `shift` | varchar(20) | Not Null | Explicit name string of the operational manufacturing shift (e.g., "Shift A"). |
| `open_week` | varchar(255) | Not Null | Calculated calendar week tracking string capturing when the ticket opened. |
| `prirority_id` | integer | **FK** | Links to `ticket_master_priority.id` enforcing action SLA parameters. |
| `issue_priority` | varchar(10) | Not Null | Text code label reflecting severity impact (e.g., "CRITICAL", "LOW"). |
| `donwtime` | integer | Nullable | Total elapsed duration of production stoppage measured in net minutes. |
| `short_desc` | varchar(50) | Not Null | High-level summary string describing the line failure signature. |
| `description` | varchar(1000) | Not Null | Long-form technical explanation detailing the machine/process failure behavior. |
| `status_id` | integer | **FK** | Links to `ticket_master_status.id` governing ticket workflow states. |
| `issue_status` | varchar(50) | Nullable | Current structural milestone lifecycle string (e.g., "OPEN", "RESOLVED"). |
| `category_id` | integer | **FK** | Links to `ticket_master_issue_category.id` mapping root infrastructure groupings. |
| `issue_cat` | varchar(50) | Nullable | Cached string value descriptive summary of the overarching systemic fault line. |
| `origin_by` | varchar(50) | Nullable | Name parameter identifying the initiating root cause operator or system. |
| `origin_team` | varchar(50) | Nullable | Department workspace group responsible for introducing or tracking the issue. |
| `owner_id` | integer | **FK** | Links to `ticket_master_user.id` denoting the engineer assigned to troubleshoot. |
| `owner_name` | varchar(50) | Nullable | Captured display name parameter of the technician currently owning the fix execution. |
| `a_category_id` | integer | **FK** | Links to `ticket_master_app_category.id` pinning application/software matrices. |
| `category` | varchar(50) | Nullable | Sub-tier structural system categorization label. |
| `sub_cat` | varchar(50) | Nullable | Deep-level granular taxonomy detail tracking exact technical failure items. |
| `rca` | varchar(1000) | Nullable | **Root Cause Analysis:** Definitive engineering diagnostic report of why the stop occurred. |
| `ica` | varchar(1000) | Nullable | **Interim Containment Action:** Quick band-aid patch applied to immediately resume line flow. |
| `pca` | varchar(1000) | Nullable | **Permanent Corrective Action:** Long-term design/process overhaul implemented to stop reuse. |
| `target_date` | date | Nullable | Hard calendar deadline assigned for delivering the permanent corrective action. |
| `closed_date` | date | Nullable | Calendar date when the issue status was systematically updated to 'CLOSED'. |
| `closed_week` | integer | Nullable | Numerical calendar week representation indicating the exact closing window milestone. |
| `resol_time` | integer | Nullable | Net metric calculating response velocity (Minutes from opening timestamp to close). |
| `remarks` | varchar(100) | Nullable | Miscellaneous validation notes, auditing records, or auxiliary engineering points. |
| `reviewed_by` | varchar(50) | Nullable | Quality Auditor or Lead Engineer name confirming verification of the patch metrics. |
| `reveiw_date` | date | Nullable | Calendar date when the audit review confirmation loop was officially submitted. |

## 🗂️ Dimension Tables Dictionary (Master Registries)

### 2. `ticket_master_user`
Anchors authentication credentials, display profile mappings, and email notification properties for plant floor users.

| Column Name | Data Type | Key / Constraint | Description |
| :--- | :--- | :---: | :--- |
| `id` | integer | **PK**, Identity | Core auto-increment serial unique key matching user entity rows. |
| `name` | varchar(50) | Not Null | Display or legal name of the employee profile (e.g., "Sushank S"). |
| `created_date` | timestamp | Default `NOW()` | System date recording when the profile was initialized. |
| `updated_date` | timestamp | Nullable | System date tracking the last modification to the profile configuration properties. |
| `is_active` | varchar(10) | Default `'ACTIVE'` | Security state flag. `'INACTIVE'` blocks API gateway connections. |
| `user_name` | varchar(50) | Nullable | Unique lowercase alphanumeric sequence used as the terminal login key. |
| `user_pwd` | varchar(255) | Nullable | Securely housed 60-character Bcrypt encrypted string hash token. |
| `email` | varchar(50) | Nullable | Corporate notification mailing target address. |
